# Pixel Art Quiz Game (像素點陣知識問答遊戲)

這是一個以 React + Vite 開發的像素風格知識問答遊戲。玩家可以輸入自己的 ID 進入遊戲，挑戰隨機生成的題目並記錄闖關成績。遊戲的題目來源與成績儲存皆由 Google Sheets 和 Google Apps Script 作為輕量級的後端資料庫支援。

## 系統需求

- Node.js (建議版本 18 以上)
- npm 或 yarn
- 一組 Google 帳號 (用於建立表單與 Apps Script)

## 安裝與執行前端專案

1. **安裝依賴套件**
   請在專案根目錄下執行：
   ```bash
   npm install
   ```

2. **設定環境變數**
   在專案根目錄下會有一個 `.env` 檔案（如果沒有請新增一個），你需要將剛才部署好的 Google Apps Script 網址填入：
   ```env
   VITE_GOOGLE_APP_SCRIPT_URL=這裡填入你的AppsScript網址
   VITE_QUESTION_COUNT=5  # 每次遊戲抽出的題數
   ```
   *註：如果 `VITE_GOOGLE_APP_SCRIPT_URL` 是空的，遊戲將會啟動「本機測試模式」（使用假資料計分）。*

3. **啟動開發伺服器**
   ```bash
   npm run dev
   ```
   伺服器啟動後，開啟瀏覽器並前往終端機顯示的網址（通常為 `http://localhost:5173`）即可開始遊戲。

4. **打包正式區塊**
   ```bash
   npm run build
   ```

## Google Sheets 後端資料庫設定指南

為了讓專案能取得題庫並記錄分數，你必須設定 Google Sheets 和 Google Apps Script。請依照下列步驟操作：

### 步驟 1：建立 Google 試算表
1. 登入 Google 雲端硬碟，新增一個空白的 Google 試算表。
2. 將這份試算表命名為「Pixel Quiz Game DB」或任何你喜歡的名稱。
3. 在左下角，會有預設的「工作表1」，請建立**兩個**工作表，名稱必須精確為以下兩個（不能有錯字或多餘空白）：
   - `題目`
   - `回答`

### 步驟 2：設定「題目」工作表
這張表用來儲存你的題庫，第一列必須是標題列：
1. **A1** 填入 `題號`
2. **B1** 填入 `題目`
3. **C1** 填入 `A`
4. **D1** 填入 `B`
5. **E1** 填入 `C`
6. **F1** 填入 `D`
7. **G1** 填入 `解答`

接著你可以從第二列開始填入你的題目（解答欄位請填入大寫的 A, B, C, 或 D）。本 README 底部有提供 10 題範例題庫可以直接複製貼上使用。

### 步驟 3：設定「回答」工作表
這張表用來記錄玩家的分數，第一列必須是標題列：
- `ID`、`闖關次數`、`總分`、`最高分`、`第一次通關分數`、`花了幾次通關`、`最近遊玩時間`

*(註：即便你不設定標題，當第一筆成績送出時，腳本也會自動幫你建立。建議可以手動建好比較好對齊。)*

## Google Apps Script 部署指南

### 步驟 1：開啟 Apps Script 編輯器
1. 在你剛剛建立好的 Google 試算表畫面中，點擊上方選單的 **「擴充功能」** > **「Apps Script」**。
2. 會開啟一個新分頁進入 Apps Script 編輯器。你可以將專案名稱重新命名為「Quiz Game API」。

### 步驟 2：貼上程式碼
1. 將專案當中的 `Code.gs` 檔案裡面所有的內容複製起來。
2. 貼到 Apps Script 編輯器中覆蓋原本的內容。
3. 點擊上方的「儲存專案」（磁碟機圖示或 `Ctrl+S` / `Cmd+S`）。

### 步驟 3：部署為網頁應用程式
1. 點擊畫面右上角的 **「部署」** > **「新增部署作業」**。
2. 點擊「選取類型」旁邊的齒輪圖示，勾選 **「網頁應用程式 (Web app)」**。
3. 表單設定：
   - 說明：(可隨便填，例如 `v1`)
   - 執行身分：選擇 **「我 (登入的電子郵件)」**
   - 誰可以存取：選擇 **「所有人 (Anyone)」** (這個非常重要，否則前端無法呼叫 API)。
4. 點擊 **「部署」**。

### 步驟 4：授權
1. 第一次部署時，系統會跳出「需要授權」的提示，點擊「授權存取」。
2. 選擇你的 Google 帳號。
3. 此時會跳出「Google 尚未驗證這個應用程式」的紅色警告。這是正常的，因為腳本是你自己寫的。點擊左下角的 **「進階」**，然後點擊最下方 **「前往 [你的專案名稱] (不安全)」**。
4. 點擊「允許」。

### 步驟 5：完成
部署完成後，畫面上會出現一串 **「網頁應用程式網址 (Web app URL)」**。
將整串網址複製下來，貼到你專案根目錄的 `.env` 檔案中的 `VITE_GOOGLE_APP_SCRIPT_URL=` 後面，就大功告成了！
